   Durum: Model KURULDU + docx UYGULANDI. base.py'de Mark enum + InlineRun
          + text-blogunda opsiyonel `runs` alani var (kanonik `text` aynen
          duruyor; runs sadece bicim tasiyan bloklarda dolar). Yalnizca
          anlamsal marklar modelleniyor: bold, italic, underline, strike,
          superscript, subscript (renk/font/highlight kozmetik -> haric).
          docx `_walk_para` bunlari w:rPr'den okuyor.
   Eksik: pptx / xlsx / html / markdown parser'lari `runs`'i HIC doldurmuyor;
          bicim bilgisi bu formatlarda hala duz metne dusuyor.
            * pptx: python-pptx run'larinda a:rPr (b/i/u) + a:t okunmali;
                    ayni Mark set'ine map'lenmeli.
            * html: <b>/<strong>/<i>/<em>/<u>/<s>/<sup>/<sub> su an "inline
                    tag" olarak yok sayilip metne akiyor; mark'a cevrilmeli.
            * markdown: **bold** / *italic* / ~~strike~~ ham metinde kaliyor;
                    parse edilip mark'a cevrilmeli (ve isaretler metinden
                    ayiklanmali).
            * xlsx: hucre-ici rich text (openpyxl CellRichText) okunabilir;
                    dusuk oncelik.
   Ek (docx): su an SADECE dogrudan w:rPr okunuyor; paragraf/karakter STILINE
          gomulu bold/italik (word/styles.xml) henuz cozulmuyor -> stille
