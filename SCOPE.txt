SCOPE
=====
What the project does: converts docx, pptx, xlsx, html, markdown, pdf files into a common
IR (ParsedDocument); structures tables, marks images, and preserves heading/list/formatting
information.

docx
----
Reads headings (real style + style mimicked via formatting), bold/italic/underline/
strikethrough/superscript-subscript formatting, lists, tables (including nested tables and
in-cell images), formulas (as approximate text), and images (including legacy and modern
formats).

pptx
----
Reads headings (real + style mimicked via formatting), bold/italic/underline/strikethrough
formatting, lists, tables, images, embedded Excel/Word objects (as previews), and slide notes.
Does not read charts/SmartArt. Does not yet read formatting inside table cells.

xlsx
----
Reads each sheet as a table, extracting merged cells and images embedded in the sheet. Reads
only the last calculated value, not the formula. Does not read charts/pivot tables, does not
read cell formatting.

html
----
Reads headings, paragraphs, lists, tables (including nested tables and in-cell images), and
bold/italic/underline/strikethrough formatting; preserves `<pre>` content verbatim. Does not
yet read formatting inside table cells.

markdown
--------
Reads headings (ATX and setext), lists, tables, images, and bold/italic/strikethrough
formatting. Does not yet read blockquotes (`>`), reference links, or inline HTML.

pdf
---
Checks whether a page is digital or scanned; reads it with a vision-language model when
necessary and verifies it against an independent source (for text, table merges, and images).
Embedded photos/diagrams are emitted as ImageBlocks with their true position at read time, on
both digital and scanned pages; a figure with no geometric counterpart is marked "unverified"
and does not silently disappear — if the VLM supplied its own bbox guess for the figure, the
crop is tightened to that guess; if it didn't, no crop is produced at all. Bold/italic is read
deterministically from the run's own font name (the PDF equivalent of reading run style in
docx): directly on digital pages (code path), and on digital pages read by the VLM (hybrid) by
aligning back to the text-layer-verified text. This information is not read on scanned pages
(scanned, no real text layer). Heading level (1-6) is ranked document-wide by font size — the
same font size maps to the same level wherever it occurs in the document, not relative to a
single page.

Other stages
------------
images/: locates every image (including ones inside table cells) at its source and stores it by
sha256; runs it through OCR with a VLM, and if meaningful, writes the corrected text into the
image's own record. Does not resolve remote (http/https) image addresses.
tables/: implemented, generates table descriptions. With the optional `tables/structure/`, can
read the table grid (cell bbox/rowspan/colspan) from a separate model (VLM, TableFormer, or an
http service — `TABLE_STRUCT_PROVIDER`); falls back to geometric table detection if not
configured.
webapp/: not yet implemented.
